Starter Pack V1 focus
=====================

This patch moves starter_village and starter_forest layout data out of C++ defaults
and into asset-authored JSON region files.

Included changes:
- RegionLayoutDatabase can load layout data from assets/regions/*.json
- RegionEcsLoader now uses travel targets and NPC ids from layout files
- OpenWorldRuntimeScene loads region layouts from assets/regions first
- assets/game.project added at the path the runtime actually loads
- starter_village.json and starter_forest.json include real layout sections

Starter region JSON format:
- id
- displayName
- scene
- safeZone
- links
- layout.playerSpawn
- layout.props
- layout.npcSpawns
- layout.gatherPoints
- layout.travelPoints
- layout.enemySpawns

How to make the game easier from here:
1) Duplicate starter_village.json or starter_forest.json
2) Rename id and displayName
3) Edit layout arrays instead of changing C++
4) Update links and start_region in assets/game.project

Recommended next version:
- data-driven scene object spawns and prefabs from region files
- dedicated NPC placement file or richer npcSpawns fields (dialogue, merchant role, quest giver role)
- editor save/export directly into region layout JSON
