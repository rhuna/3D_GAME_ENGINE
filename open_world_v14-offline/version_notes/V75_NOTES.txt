v75 - fully content-driven starter region pass

What changed:
- RegionLayoutDatabase can now load region layout data from assets/regions/*.json.
- Runtime scenes now prefer file-backed layouts and only fall back to hardcoded defaults when no JSON layouts are found.
- Starter scaffold now generates starter_village.json, starter_forest.json, and _template_region.json with a layout block.
- Added starter region JSON files directly into assets/regions for easier adoption in existing projects.

Recommended workflow:
- Copy assets/regions/_template_region.json.
- Rename id, displayName, and scene.
- Edit layout.playerSpawn, props, npcSpawns, gatherPoints, travelPoints, and enemySpawns.
- Set start_region in assets/game.project.
