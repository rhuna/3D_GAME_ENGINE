Apply these permanent-fix changes to RegionEcsLoader.cpp:

1) Do not return early if RegionLayoutDatabase has no layout for the region.

Current code:
    const RegionLayout* layout = layouts.Find(regionId);
    if (!layout) {
        Logger::Warn("REGION", "RegionEcsLoader could not find layout for region: " + regionId);
        return;
    }

Change to:
    const RegionLayout* layout = layouts.Find(regionId);
    if (!layout) {
        Logger::Warn("REGION", "No fallback layout for region: " + regionId + ". Continuing with scene-driven region content only.");
    }

2) Guard all layout-driven loops with if (layout) { ... }.
This includes props, gatherPoints, npcSpawns, travelPoints, and layout enemy spawns.

3) Replace hardcoded travel destinations for scene-tagged travel markers with content-driven links from RegionDefinition.links.

Suggested pattern:
    if (const RegionDefinition* region = pipeline.regions.Find(regionId)) {
        // Use region->links to assign TravelTriggerComponent destinations.
    }

4) Keep RegionLayoutDatabase as fallback-only, not as the source of truth.
